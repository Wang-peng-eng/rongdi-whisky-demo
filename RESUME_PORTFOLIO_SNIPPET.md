# 简历与面试用项目描述

## 简历版

荣帝威士忌内容体系中心 Demo  
项目链接：https://rongdi-whisky-demo-vercel.vercel.app/

基于国产威士忌品牌「荣帝」真实资料，独立搭建一个品牌内容体系展示 Demo。项目将品牌故事、五款产品信息、FAQ 知识库与内容生成模拟整合为可体验的网站，用于展示品牌内容资产结构化、产品卖点拆解、用户教育和跨平台内容转化能力。技术上使用 React + Tailwind CSS + Vite，无后端，mock 数据驱动，并部署至 Vercel。

核心贡献：

- 将品牌背景拆解为起源、定位、理念、关键词等可复用叙事模块。
- 建立产品内容模型，统一管理风味、酒龄、桶型、酒精度、场景和内容角度。
- 设计 FAQ 知识库，将消费者疑问转化为客服、直播和社媒运营素材。
- 设计内容生成模拟流程，展示公众号、小红书、短视频和直播话术的差异化表达。
- 完成品牌官网级 UI 升级，使作品从工具型 Demo 转向可展示的商业作品集页面。

## 面试 30 秒介绍

这是我做的一个品牌内容体系 Demo，主题是荣帝威士忌。我的重点不是做一个普通官网，而是展示如何把品牌资料变成内容资产。我把它拆成品牌故事、产品体系、FAQ 知识库和内容生成模拟四个模块，让品牌叙事、产品卖点、用户问题和平台内容之间形成连接。这个项目主要体现我的内容结构化能力、内容运营能力，以及把内容方法产品化表达的能力。

## 面试 1 分钟介绍

我做「荣帝威士忌内容体系中心 Demo」的出发点是：很多品牌有大量资料，但资料本身不能直接等于运营能力。运营真正需要的是一套能复用、能分发、能降低用户理解成本的内容体系。

所以我把荣帝的品牌资料拆成四层：第一层是品牌故事，讲清楚青岛山海、崂山水源和中国威士忌风土；第二层是产品体系，把五款酒按照风味、桶型、酒精度和场景统一建模；第三层是 FAQ 知识库，用来承接消费者教育和客服答疑；第四层是内容生成模拟，把产品信息转化成公众号、小红书、短视频和直播话术。

这个作品希望证明的是，我不只是能写单篇内容，而是能从品牌、产品、用户问题和平台分发四个角度搭建内容系统。
