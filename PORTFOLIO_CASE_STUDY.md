# 荣帝威士忌内容体系中心 Demo

## 项目定位

这是一个面向求职展示的品牌内容体系 Demo，主题为「荣帝威士忌」。项目不是传统后台工具，也不是单纯品牌官网，而是一个用于展示内容运营方法论的品牌内容资产系统。

项目通过品牌故事、产品体系、FAQ 知识库与内容生成模拟四个模块，呈现我对「品牌内容体系搭建」的理解：如何把分散的品牌资料、产品信息、直播话术和消费者疑问，转化为可管理、可复用、可分发的内容资产。

## 项目背景

威士忌品牌的内容运营常见问题是：品牌故事分散、产品卖点不成体系、客服与直播话术重复生产、不同平台内容风格不统一。对于一个正在建立市场认知的国产威士忌品牌来说，内容不仅是传播材料，更是用户理解品牌、产品和消费场景的路径。

因此，本项目将「内容体系」作为核心，而不是简单做一个产品展示页。项目重点解决三个问题：

1. 如何把品牌真实信息结构化，形成可展示的品牌叙事。
2. 如何把五款威士忌产品拆解成统一的产品内容模型。
3. 如何把 FAQ、直播话术和平台内容生成连接起来，提高内容运营复用效率。

## 我的角色

在这个 Demo 中，我承担了内容产品经理与内容运营设计者的角色，负责从资料理解、信息架构、内容拆解到前端 Demo 呈现的完整流程。

主要工作包括：

- 梳理品牌背景资料，提炼「青岛山海」「崂山水源」「自发酵自蒸馏自陈酿」「中国威士忌被世界看见」等核心叙事。
- 将产品信息统一拆解为风味、酒龄、桶型、酒精度、适饮场景和内容角度。
- 将常见问题转化为 FAQ 知识库，用于客服、直播、社媒评论区和私域答疑。
- 设计内容生成模拟模块，展示从产品资料到公众号、小红书、短视频、直播话术的跨平台转化逻辑。
- 使用 React + Tailwind CSS 搭建可本地运行并可公网访问的交互 Demo。

## 内容体系方法

本项目采用「品牌叙事层 - 产品表达层 - 知识问答层 - 内容分发层」的结构。

### 1. 品牌叙事层

品牌故事页不是简单堆文案，而是将真实资料拆成三个核心叙事单元：

- 起源：来自青岛崂山山海之间的蒸馏厂。
- 定位：中国风土单一麦芽威士忌。
- 理念：时间、自然、工艺共同完成酒体表达。

这样处理的价值是：品牌故事可以被反复复用到官网、公众号、招商材料、直播开场、产品详情页和媒体稿中。

### 2. 产品表达层

产品体系页将五款威士忌统一纳入一个产品内容模型：

- 产品名称
- 酒精度
- 酒款类型
- 桶型信息
- 成熟方式
- 风味关键词
- 饮用或传播场景
- 内容传播角度

这种结构能让不同产品之间形成清晰对比，也方便运营团队在写内容时快速抓住每款酒的差异化。

### 3. 知识问答层

FAQ 知识库围绕消费者真实疑问设计，包括：

- 国产威士忌与进口威士忌如何比较
- 不同酒精度代表什么
- 开瓶后如何保存
- 出现沉淀物是否正常
- Palo Cortado 桶型为什么值得讲

FAQ 不只是售后问答，而是内容运营的基础素材。每个问题都可以继续延展成小红书笔记、直播答疑、短视频脚本或客服标准话术。

### 4. 内容分发层

内容生成模拟页展示的是「内容资产如何被复用」。输入产品名称后，系统会模拟输出：

- 公众号标题
- 小红书文案
- 短视频脚本
- 直播话术

这个模块的重点不在于真正替代 AI 工具，而是展示内容运营的底层逻辑：同一份产品信息，需要根据不同平台的用户心智重新表达。

## 信息架构

网站保留五个核心入口：

1. 首页：品牌内容体系入口，建立第一屏品牌感。
2. 品牌故事页：承接品牌起源、定位、理念和关键词。
3. 产品体系页：展示五款威士忌的产品矩阵和内容拆解。
4. FAQ 知识库页：沉淀消费者教育与客服话术。
5. 内容生成模拟页：展示内容从产品资产到平台表达的转化能力。

这个结构对应的是一个内容运营团队的真实工作流：先定义品牌叙事，再管理产品资产，再沉淀问题知识，最后分发到各个平台。

## 视觉与交互策略

项目视觉从「工具型后台」调整为「品牌官网级作品展示」：

- 首页采用大标题、品牌定位、产品视觉和主 CTA，强化第一屏冲击力。
- 页面整体增加留白，降低信息压迫感。
- 卡片使用柔和圆角、轻阴影和 hover 动效，呈现 SaaS 产品级质感。
- 使用模块化布局，让用户能快速理解每个内容资产模块的作用。
- 产品图片保留原始视觉，不做二次 AI 重绘，避免破坏真实品牌信息。

整体参考 Notion 的结构感、Stripe 的层级感和 Apple 官网的视觉留白，但没有照搬任何模板。

## 技术实现

技术栈：

- React
- Tailwind CSS
- Vite
- mock 数据
- 无后端
- Vercel 部署

实现方式：

- 使用 `src/content.js` 集中管理品牌故事、产品信息、FAQ 与内容生成模板。
- 使用组件化方式拆分页面模块，保证展示逻辑清晰。
- 使用 Tailwind 完成响应式布局、卡片样式、动效和视觉层级。
- 使用 Vercel 部署为公网可访问作品链接。

## 项目亮点

### 亮点一：不是页面堆砌，而是内容资产结构化

这个项目的核心不是把品牌资料搬到页面上，而是将资料拆成可复用的内容资产。品牌故事、产品信息、FAQ 和平台文案都能互相连接，体现内容体系搭建能力。

### 亮点二：产品内容模型统一

五款威士忌被放入同一套内容框架中，既保留各自差异，又能形成统一的产品矩阵。这种方式适合品牌官网、招商手册、直播间培训和社媒内容规划。

### 亮点三：FAQ 被设计为运营资产

FAQ 不只是回答问题，而是帮助品牌降低用户理解成本。每个 FAQ 都能继续转化为教育内容、销售话术和评论区回复。

### 亮点四：展示跨平台内容转化能力

内容生成模拟模块体现了从产品信息到不同平台表达的转化能力。公众号强调叙事，小红书强调种草，短视频强调镜头结构，直播话术强调即时转化。

### 亮点五：Demo 具备真实展示价值

项目已完成公网部署，可直接作为求职作品链接展示。相比静态 PPT，它能让面试官直接体验信息结构、内容模块和交互逻辑。
